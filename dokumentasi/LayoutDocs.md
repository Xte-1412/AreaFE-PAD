# 📑 Layout Dokumentasi SIPELITA

Halaman ini adalah **layout/hub dokumentasi** untuk kebutuhan tim FE/BE/PM.
Gunakan struktur ini agar dokumen konsisten, mudah dicari, dan gampang dipahami pembaca publik maupun internal.

---

## 1) Dokumen yang Tersedia Saat Ini

| Jenis Dokumen | Nama File | Status | Catatan |
|---|---|---|---|
| Layout/HUB | `LayoutDocs.md` | ✅ Tersedia | Navigasi & standar dokumentasi |
| Progress Public | `Progress PAD2.md` | ✅ Tersedia | Update progress per sprint |
| Sprint Report (Single File) | `Sprint-Report.md` | ✅ Tersedia | Satu file untuk update berkala setiap sprint |
| Tech Plan Template | `template/TechPlan-Template.md` | ✅ Tersedia | Template rencana teknis |
| Testing Template | `template/Testing-Sprint-Template.md` | ✅ Tersedia | Template hasil testing sprint |
| Sprint Report Template | `template/Sprint-Report-Template.md` | ✅ Tersedia | Template laporan sprint publik |
| Release Notes Template | `template/Release-Notes-Template.md` | ✅ Tersedia | Template changelog rilis |

---

## 2) Cara Pakai Singkat

1. Untuk laporan sprint berkala publik, gunakan file tetap: `Sprint-Report.md`.
2. Update section sprint aktif (hasil, dampak, status) saat sprint selesai.
3. Untuk dokumen teknis lain, copy template sesuai kebutuhan.
4. Tambahkan entry di log update dokumen.

---

## 3) Konvensi Penamaan Dokumen

- Tech plan: `TechPlan-[NamaFitur].md`
- Testing sprint: `Testing-Sprint1.md`, `Testing-Sprint2.md`, dst.
- Sprint report publik berkala: `Sprint-Report.md` (single file).
- Release notes: `Release-Notes-vX.Y.Z.md`
